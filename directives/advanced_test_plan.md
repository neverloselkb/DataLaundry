# 심화 NLP 테스트 시나리오 정의

## 1. 테스트 목적
현재 `processors.ts`에 구현된 NLP 엔진이 복잡한 사용자 요구사항과 예외 상황을 얼마나 정확하게 처리하는지 검증하고, 로직적 한계를 파악합니다.

## 2. 심화 테스트 케이스 (Advanced Cases)

| ID | 시나리오 | 대상 데이터 및 프롬프트 | 기대 결과 |
| :--- | :--- | :--- | :--- |
| **A-1** | **주소 계층 분리** | "서울시 강남구 역삼동" -> `추출/분리` 시나리오 | 시/도, 구/군, 동읍면 각각 정확히 추출 |
| **A-2** | **복합 패턴 치환** | "대괄호 [포함] 텍스트 제거하고 지워줘" | 괄호와 그 내용이 모두 삭제됨 |
| **A-3** | **부분 일치 타겟팅** | "연락처 컬럼 앞에 [VIP] 붙여줘" | 컬럼명이 'tel'인 경우에도 '연락처' 키워드로 타겟팅 성공 |
| **A-4** | **복수 규칙 결합** | "대문자로 바꾸고 뒤에 _FIXED 붙여" | 순차적 처리 (ABC -> ABC_FIXED) |
| **A-5** | **조건부 값 치환** | "1000이 넘는 값은 'VIP'로 바꿔" | 숫자 패턴 매칭 및 치환 |
| **A-6** | **이메일 도메인 추출** | "이메일에서 도메인만 남겨줘" | @ 뒷부분만 추출 성공 |
| **A-7** | **날짜 형식 강제 변경** | "날짜를 yyyymmdd 붙여서 포맷으로" | 구분자 제거 및 8자리 형식 통일 |
| **A-8** | **불필요한 수식어 제거** | "회사 이름에서 주식회사랑 (주) 싹다 지워" | 수식어 제거 및 trim 적용 |

## 3. 테스트 데이터 세트
- 다양한 노이즈가 섞인 실무형 데이터 구성 (주소, 이메일, 특수문자, 직함 등 포함)
