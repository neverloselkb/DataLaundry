# 기본 워크플로우 지시문

이 문서는 `data-clean-ai` 프로젝트의 핵심 운영 절차를 정의합니다.

## 1. 데이터 정제 프로세스
1. 사용자가 정제할 데이터를 업로드합니다.
2. `execution/` 내의 적절한 스크립트를 사용하여 데이터를 처리합니다.
3. 처리 중 발생하는 중간 파일은 `.tmp/`에 저장합니다.
4. 최종 결과물은 사용자에게 제공하거나 클라우드 저장소에 업로드합니다.

## 2. 도구 추가 및 수정
1. 새로운 비즈니스 로직이 필요한 경우 `execution/` 폴더에 TypeScript/Node.js 스크립트를 생성합니다.
2. 스크립트는 결정론적이어야 하며 모듈화되어 있어야 합니다.
3. 새로운 도구를 추가한 후에는 이 지시문을 업데이트하여 해당 도구의 사용법을 명시합니다.

## 3. 오류 처리
1. 실행 중 오류가 발생하면 `execution/` 스크립트의 로그를 확인합니다.
2. 오류의 원인이 환경 설정인 경우 `.env`를 확인합니다.
3. 로직 오류인 경우 스크립트 수정 후 테스트를 거쳐 반영합니다.
## 4. 시스템 점검 방법
1. 모든 기능의 무결성을 확인하려면 `execution/full_system_check.ts` 스크립트를 실행합니다.
2. 실행 명령어: `npx tsx execution/full_system_check.ts`
3. 새로운 기능을 추가할 때는 반드시 해당 스크립트에 테스트 케이스를 추가하여 회귀 테스트를 수행합니다.
